## Frontend MVP (React Native + Expo + Web) под ТЗ

Цель: один фронтовый проект (мобилка iOS/Android + веб) на React Native/Expo, реализующий все пункты ТЗ в формате MVP поверх нашего REST API.

ТЗ (кратко): единая панель подписок, автосбор (банки/почта), аналитика, анализ использования, прогноз, умные уведомления, рекомендации альтернатив, быстрая отмена/пауза.

---

## 1. Стек и структура проекта

- **Стек**:
  - React Native + Expo (Managed Workflow).
  - React Native Web (Expo Web build) → веб-приложение.
  - TypeScript.
  - Expo Router / React Navigation.
  - React Query (общение с API).
  - Zustand/Redux Toolkit (auth, UI-состояние).

- **Структура (пример)**:

```text
app/
  (auth)/
    onboarding.tsx
    login.tsx
    register.tsx
  (main)/
    dashboard.tsx
    subscriptions/
      index.tsx        // список подписок
      [id].tsx         // деталь подписки
    analytics/
      index.tsx
    notifications/
      index.tsx
    recommendations/
      index.tsx
    profile/
      index.tsx
shared/
  components/
  api/
  hooks/
  store/
  theme/
```

## 2. Соответствие пунктам ТЗ → реализации на фронте (MVP)

### 2.1. Централизованное отображение всех активных подписок

- **Экран(ы)**:
  - `Dashboard` (`(main)/dashboard.tsx`):
    - Заголовок: «Мои подписки».
    - Краткое summary: количество подписок, траты в месяц/год (данные с `/analytics/summary`).
    - Диаграмма по категориям (простая круговая/бар-чарт).
    - Кнопка/строка поиска.
  - `SubscriptionsList` (`(main)/subscriptions/index.tsx`):
    - Список карточек подписок (данные с `/subscriptions`).
    - Фильтры по статусу/категории.
    - FAB/кнопка `+` → форма добавления подписки.

- **Связь с бэком**:
  - `GET /api/v1/subscriptions`.
  - `GET /api/v1/analytics/summary`.

### 2.2. Автоматический сбор данных (банки и/или почта)

- **Экран(ы)**:
  - `SourcesScreen` (часть онбординга или экран в `profile`):
    - Блок «Банки» (Сбер, Тинькофф, ЮMoney и т.д.).
    - Блок «Почта» (Mail.ru, Yandex, Gmail).
    - Для каждого — статус (`Подключено`/`Не подключено`).
    - Кнопки:
      - `Подключить` → в MVP просто создаём источник и считаем его подключённым.
      - `Загрузить выписку/чеки` → открываем файловый диалог и шлём файл на `/sources/:id/upload`.

- **MVP-ограничение**:
  - Вместо реального OAuth:
    - Показываем модалку «В реальном продукте здесь будет OAuth к банку/почте, в MVP — загрузка выписки/демо-файла».
    - После выбора «подключить» — сразу ставим статус `Подключено`.

- **Связь с бэком**:
  - `GET /api/v1/sources`.
  - `POST /api/v1/sources`.
  - `POST /api/v1/sources/:id/mark-connected`.
  - `POST /api/v1/sources/:id/upload` (upload контрол).

### 2.3. Визуализация аналитики по затратам

- **Экран**:
  - `AnalyticsScreen` (`(main)/analytics/index.tsx`):
    - Табы/переключатель периода: `Месяц` / `Год`.
    - Карточка «Траты за период» с динамикой.
    - Диаграмма по категориям.
    - Список категорий с суммой и процентом.
    - Блок «ТОП-5 подписок по тратам».

- **Связь с бэком**:
  - `GET /api/v1/analytics/summary`.
  - `GET /api/v1/analytics/categories`.
  - `GET /api/v1/analytics/services`.

### 2.4. Анализ использования подписок

- **Экран**:
  - `SubscriptionDetail` (`(main)/subscriptions/[id].tsx`):
    - Данные подписки (`/subscriptions/:id`).
    - История платежей (транзакции).
    - Блок «Использование»:
      - Счётчик дней/событий использования за месяц.
      - Кнопка `Отметить, что пользовался сегодня`.

- **Связь с бэком**:
  - `GET /api/v1/subscriptions/:id` (включаем usage summary).
  - `POST /api/v1/subscriptions/:id/usage`.

### 2.5. Прогноз трат на год вперёд

- **Где показываем**:
  - `AnalyticsScreen`:
    - Отдельный блок «Прогноз на год»:
      - `Ожидаемые траты на год`.
      - `Экономия при отключении слабоиспользуемых подписок`.

- **Связь с бэком**:
  - `GET /api/v1/forecast/year`.

### 2.6. Настройка и отправка умных уведомлений

- **Экран**:
  - `NotificationsSettings` (`(main)/notifications/index.tsx` — вкладка «Настройки» + «История»):
    - Тумблеры:
      - Предстоящее списание.
      - Повышение цены.
      - Низкая активность.
      - Выгодные предложения.
    - Для каждого — чекбоксы каналов (`Push`, `Email`) — в MVP можно сделать только UI + хранение на бэке.
    - Вторая вкладка: «История уведомлений» — список уведомлений.

- **Поведение**:
  - При первом включении типа уведомления — показываем попап «Что вы будете получать».
  - Внутри приложения — отображаем уведомления как список/баннеры (без реального push).

- **Связь с бэком**:
  - `GET /api/v1/notifications/settings`.
  - `PUT /api/v1/notifications/settings`.
  - `GET /api/v1/notifications`.
  - `PUT /api/v1/notifications/:id/read`.

### 2.7. Поиск и рекомендации альтернатив

- **Экраны**:
  - В `Dashboard`/`SubscriptionsList`:
    - Строка поиска по названию сервиса/категории (фильтрация на клиенте + запрос к API).
  - `RecommendationsScreen` (`(main)/recommendations/index.tsx`):
    - Фильтры: категория, цена.
    - Список карточек альтернативных сервисов.
  - В `SubscriptionDetail`:
    - Блок «Альтернативы» со ссылкой `Посмотреть все` → переход в `RecommendationsScreen` с предзаполненным фильтром.

- **Связь с бэком**:
  - `GET /api/v1/recommendations`.
  - `GET /api/v1/recommendations/subscription/:id`.

### 2.8. Быстрая отмена или приостановка подписки

- **Экран**:
  - `SubscriptionDetail`:
    - Кнопка `Отменить подписку`.
    - Кнопка `Поставить на паузу`.

- **Флоу отмены (MVP)**:
  1. Пользователь нажимает `Отменить подписку`.
  2. Фронт вызывает `POST /api/v1/subscriptions/:id/cancel-intent`.
  3. Бэк возвращает:
     - `cancel_url` (если есть).
     - `support_email`.
     - `email_template` — текст письма.
  4. Фронт показывает модалку:
     - Кнопка `Перейти в личный кабинет` (открыть `cancel_url` в браузере/вебвью).
     - Кнопка `Написать в поддержку` (открыть системный почтовый клиент с темой и текстом).
  5. Доп.кнопка `Я уже отменил` → `POST /api/v1/subscriptions/:id/confirm-cancel` → обновляем статус в UI.

- **Флоу паузы (MVP)**:
  1. Пользователь нажимает `Поставить на паузу`.
  2. Модалка:
     - Выбор периода (1/3 месяца).
     - Чекбокс «Только для аналитики» (не лезем в реальный сервис).
  3. `POST /api/v1/subscriptions/:id/pause`.
  4. В UI показываем статус `Приостановлена`, меняем отображение в аналитике (вытаскивается из бэка).

## 3. Навигация и layout (мобильное + веб)

### 3.1. Мобильное приложение (iOS/Android)

- Нижняя таб-бар навигация:
  - `Подписки` (Dashboard+список).
  - `Аналитика`.
  - `Рекомендации`.
  - `Профиль/Настройки`.

- FAB на экране подписок:
  - Кнопка `+` → форма добавления подписки (модалка/отдельный экран).

- Уведомления:
  - Пока только in-app (список и баннеры).

### 3.2. Веб-приложение

- Левое боковое меню:
  - `Дашборд`.
  - `Подписки`.
  - `Аналитика`.
  - `Рекомендации`.
  - `Уведомления`.
  - `Настройки/Источники`.

- Layout:
  - На десктопе — 2 колонки:
    - Слева список подписок.
    - Справа — детали/аналитика/графики.
  - На мобиле (через адаптив) — экраны стеком (как в нативе).

## 4. API-клиент и состояние

- **API-клиент**:
  - `shared/api/client.ts` — базовый клиент с JWT (хранится в Zustand/Redux).
  - Модули:
    - `shared/api/auth.ts`
    - `shared/api/subscriptions.ts`
    - `shared/api/analytics.ts`
    - `shared/api/notifications.ts`
    - `shared/api/recommendations.ts`
    - `shared/api/sources.ts`
  - Для каждого модуля — React Query-хуки:
    - `useSubscriptionsQuery`, `useCreateSubscriptionMutation`, и т.д.

- **Состояние**:
  - `authStore`: пользователь, токен, статус (logged in/logged out).
  - `uiStore`: пройден ли онбординг, выбранный период (`month`/`year`).
  - Остальные данные — через React Query (серверный стейт).

## 5. MVP-объём фронта под показ жюри

- Онбординг + регистрация/логин.
- Экран подключения источников (банки/почта) с мок-подключением и загрузкой выписки.
- Дашборд + список подписок + деталь подписки.
- Аналитика с базовым графиком и блоком прогноза.
- Экран уведомлений (настройки + история) с in-app уведомлениями.
- Экран рекомендаций и блок альтернатив в детали подписки.
- Флоу быстрой отмены и паузы как описано выше.
