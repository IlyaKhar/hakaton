## Название продукта и цель

Сервис управления подписками (мобилка iOS/Android + веб).  
Цель: собрать все платные подписки пользователя в одном месте, дать аналитику трат, помочь не переплачивать и быстро отменять/приостанавливать ненужное.

## Ключевые сценарии (для UX)

1. **Онбординг и регистрация**
   - Пользователь заходит первый раз → приветственный экран с коротким объяснением ценности сервиса.
   - Выбор типа авторизации: `Войти по номеру телефона`, `Войти по почте`, `Продолжить через Google/Apple`.
   - После авторизации — базовые разрешения: согласие на обработку данных и доступ к платежам/почте (подробно описать, что именно читаем и зачем).

2. **Подключение источников данных о подписках**
   - Экран выбора источников: `Банки`, `Почта`, `Ручной ввод`.
   - Блок `Банки`: карточки Сбербанк, Тинькофф, ЮMoney, прочие (поиск по банку).
     - Тап по банку → экран с описанием, что именно будет делаться (чтение истории операций, поиск регулярных платежей, безопасность).
     - Кнопка `Подключить банк` → переход на OAuth/веб-форму банка (внешний браузер/вебвью).
     - Статус подключения: `Подключено`, `Ошибка`, `Ожидание`, `Требуется обновить доступ`.
   - Блок `Почта`: Mail.ru, Yandex, Gmail.
     - Тап по почтовому сервису → описание, что читаем только письма с чеками/уведомлениями о списаниях.
     - Кнопка `Дать доступ к почте`.
   - `Ручной ввод`: кнопка `Добавить подписку вручную` → форма создания подписки.

3. **Главная панель подписок (Дашборд)**
   - Верхняя часть экрана:
     - Заголовок: `Мои подписки`.
     - Переключатель периода: `Месяц`, `Год`, `Все время`.
     - Краткий summary: `Всего подписок: N`, `Траты в месяц: X ₽`, `Экономия по отменённым: Y ₽`.
   - Средний блок — визуализация:
     - Круговая диаграмма по категориям (стриминги, софт, доставка, облако, другое).
     - Под диаграммой легенда по цветам + сумма по категории.
   - Нижний блок — список подписок:
     - Карточка подписки (одна строка):
       - Иконка сервиса (Netflix, Spotify, Яндекс Плюс и т.п.).
       - Название подписки.
       - Цена (в период: /мес, /год).
       - Дата следующего списания.
       - Статус: `Активна`, `Пробный период`, `Приостановлена`, `Отменена`.
       - Индикатор `Низкая активность` (если по данным пользователя редко используется).
     - Тап по карточке → экран детали подписки.
   - Фиксированная кнопка (FAB в мобилке, кнопка в правом верхнем углу на вебе): `Добавить подписку`.

4. **Экран детали подписки**
   - Шапка:
     - Логотип сервиса, название.
     - Чипы: категория, статус, тип списания (`ежемесячно`, `ежегодно`).
   - Основной блок:
     - Текущий тариф и стоимость.
     - Следующее списание: дата + сумма.
     - История платежей в виде списка (дата, сумма, источник — банк/карта/счёт).
   - Блок `Использование`:
     - Для стримингов: контрольные поля (по данным от пользователя): `Просмотрено фильмов за месяц`, `Дней с активным использованием`.
     - Для софта: `Дней использования в месяце`, `Часов в неделю`.
     - Для доставок: `Количество заказов в месяц`.
     - Пользователь может отмечать активность: кнопка `Отметить, что пользовался сегодня` / чеклист активности.
   - Блок `Аналитика по подписке`:
     - График трат по месяцам.
     - График использования vs стоимость (чтобы показать, дорого/дёшево относительно реального использования).
   - Блок `Рекомендации`:
     - Список альтернативных сервисов с отметкой «экономия до X ₽/год».
     - Кнопка `Сравнить тарифы`.
   - Нижние основные CTA-кнопки:
     - `Отменить подписку` (primary, красная/акцентная).
     - `Приостановить` (secondary).
     - `Изменить тариф` (link/button).

5. **Экран аналитики (общие траты и прогноз)**
   - Переключатель вкладок: `Подписки` | `Аналитика` | `Уведомления` (табы внизу или сверху).
   - Вкладка `Аналитика`:
     - Карточка `Траты за месяц`: сумма, динамика к прошлому месяцу (`+10%`, `-5%`).
     - Карточка `Траты за год`: сумма, динамика к прошлому году (если есть данные).
     - График по месяцам (линейный/столбчатый).
     - Разбивка по категориям: список категорий с суммой и процентом.
     - Разбивка по сервисам: топ-5 самых дорогих подписок.
   - Блок `Прогноз на год`:
     - Карточка: `Ожидаемые траты на год: Z ₽`.
     - Подробности:
       - Сколько из этого — текущие подписки.
       - Сценарий экономии: если отменить низкоиспользуемые подписки → `экономия до K ₽/год`.

6. **Экран уведомлений и настроек**
   - Список включённых типов уведомлений:
     - `Предстоящее списание` — тумблер включено/выключено, настройка за сколько дней: `1`, `3`, `7`.
     - `Повышение цены` — вкл/выкл.
     - `Низкая активность в сервисе` — вкл/выкл.
     - `Новые выгодные предложения` — вкл/выкл.
   - Для каждого типа — возможность настроить канал:
     - Чекбоксы: `Push`, `Email`, `Telegram-бот` (если реализуем).
   - Экран истории уведомлений:
     - Список событий: дата, тип, по какой подписке, статус прочтения.

7. **Поиск и рекомендации**
   - В дашборде — строка поиска: `Найти подписку или сервис`.
     - Поиск по названию сервиса, категории, тегам.
   - Отдельная вкладка/экран `Рекомендации`:
     - Фильтры: категория (стриминги, софт, доставка и т.д.), цена/месяц, регион.
     - Карточки сервисов:
       - Название, цена, краткое описание.
       - Чип `Рекомендуемая альтернатива Netflix`, `Экономия до X ₽/год`.
       - Кнопка `Перейти на сайт сервиса`.

8. **Быстрая отмена / приостановка подписки**
   - Сценарий отмены:
     - Пользователь нажимает `Отменить подписку` в деталях.
     - Модалка:
       - Заголовок: `Как будем отменять?`
       - Варианты:
         - `Перейти в личный кабинет сервиса` — кнопка ведёт на URL управления подпиской (если известен).
         - `Создать письмо в поддержку` — кнопка формирует шаблон письма (тема, текст, адрес) и открывает почтовый клиент.
       - Дополнительно: текст о том, что сам сервис не может 100% гарантировать отмену, если её не подтвердили у провайдера.
   - Сценарий приостановки:
     - Кнопка `Приостановить`.
     - Модалка:
       - Выбор периода: `1 месяц`, `3 месяца`, `До конца года`.
       - Вопрос: `Эта приостановка только для аналитики или вы реально приостанавливаете в сервисе?`
         - Радио-кнопки:
           - `Только пометить как приостановленную у нас` (мы не трогаем реальный сервис).
           - `Помочь приостановить в сервисе` (такой же сценарий, как при отмене: ссылка/письмо).

## Перечень основных экранов (для дизайнера)

- **Онбординг**: 2–3 скрина с основными выгодами, кнопка `Продолжить`.
- **Регистрация / Вход**: формы, соц. авторизация.
- **Экран подключения источников**:
  - Списки банков и почтовых сервисов, статус интеграции, CTA `Подключить`.
- **Главный дашборд подписок**:
  - Подписки, фильтры, поисковая строка, диаграмма по категориям.
- **Деталь подписки**:
  - Вся информация по конкретной подписке, кнопки управления (отмена, пауза, изменение тарифа).
- **Аналитика**:
  - Графики, карточки с суммами, переключение периодов.
- **Прогноз трат**:
  - Отдельный блок/подэкран с визуализацией прогноза и сценариями экономии.
- **Уведомления и настройки**:
  - Управление типами уведомлений и каналами доставки.
- **Рекомендации и поиск альтернатив**:
  - Каталог альтернативных сервисов с фильтрацией и сравнением.

## Логика по кнопкам и действиям

- **Кнопка `Подключить банк` / `Подключить почту`**
  - Открывает внешний OAuth/вебвью.
  - После успешного подключения — показываем статус `Подключено` и запускаем фоновый импорт транзакций/писем.
  - При ошибке — статус `Ошибка`, кнопка `Повторить`.

- **Кнопка `Добавить подписку`**
  - Открывает форму:
    - Название сервиса (автодополнение по справочнику).
    - Категория.
    - Периодичность списания (месяц/год/другое).
    - Сумма.
    - Дата следующего списания.
    - Источник оплаты (карта/банк/кошелёк).
  - После сохранения — подписка появляется в списке и учитывается в аналитике/прогнозе.

- **Кнопка `Отметить использование` (в деталях подписки)**
  - При нажатии фиксирует факт использования за сегодня.
  - Можно сделать простым тумблером «пользовался сегодня/нет» или чеклистом активности.
  - Эти данные идут в блок `Аналитика использования` и в расчёт эффективности подписки.

- **Кнопка `Отменить подписку`**
  - Открывает модалку с выбором способа отмены (ссылка в личный кабинет или письмо).
  - При выборе `Письмо` — формируем готовый текст с:
    - Приветствием.
    - Данной подписью пользователя (ФИО/почта).
    - Запросом на отмену подписки с указанием тарифа/аккаунта.
  - Внутри приложения подписка может помечаться как `В процессе отмены`.

- **Кнопка `Приостановить`**
  - Открывает модалку выбора периода и типа приостановки (только аналитика или попытка реальной паузы).
  - Если только аналитика: меняем статус подписки на `Приостановлена` и исключаем из прогноза на этот период.
  - Если реальная пауза: аналогично сценарию с отменой (ссылка/письмо).

- **Кнопка `Сравнить тарифы`**
  - Открывает экран сравнения:
    - Текущий сервис vs альтернативы.
    - Столбцы: `Цена/мес`, `Цена/год`, `Ограничения`, `Плюсы/минусы`.
  - CTA на каждой альтернативе: `Перейти на сайт`.

- **Переключатели периода (`Месяц` / `Год`)**
  - Меняют состояние графиков, суммы в summary и карточки подкручиваются под период (например, пересчёт годовых подписок в месячную стоимость).

- **Тумблеры уведомлений**
  - Управляют настройками на сервере: включать/выключать конкретный тип пушей/писем.
  - При первом включении типа уведомления— показываем небольшое пояснение (tooltip/модалка), что именно и как будет присылаться.

## Особенности для мобильного и веба

- **Мобильное приложение**
  - Нижнее таб-меню: `Подписки`, `Аналитика`, `Рекомендации`, `Профиль`.
  - FAB-кнопка `+` на экране подписок для быстрого добавления.
  - Уведомления — пуши по умолчанию, Email/Telegram — как дополнительные опции.

- **Веб-приложение**
  - Левое боковое меню: `Дашборд`, `Подписки`, `Аналитика`, `Рекомендации`, `Уведомления`, `Настройки`.
  - Более детальные таблицы и графики (можно больше информации на экране).
  - Возможность одновременно видеть список подписок и график (layout 2 колонки).

## Технические пометки для дизайнера

- Важно явно показывать статусы:
  - Для источников данных: `Подключено`, `Требуется обновление`, `Ошибка`.
  - Для подписок: `Активна`, `Пробный период`, `Приостановлена`, `Отменена`, `В процессе отмены`.
- Важны три уровня приоритета уведомлений в визуале:
  - Критические (списание завтра, сильное повышение цены).
  - Важные (низкая активность, высокая годовая трата).
  - Информационные (новые предложения, скидки).
- Во всех местах, где можем «автоматически что-то сделать», явно пояснять:
  - Что именно делаем.
  - Что сервис не списывает деньги сам, а только анализирует/подсказывает.
- Цветовые коды:
  - Позитив: экономия, отмена ненужного — зелёный.
  - Опасность/внимание: большие траты, скорое списание — оранжевый/красный.
  - Нейтральные статусы — серый/синий.

## Что важно донести визуально

- Пользователь всегда должен понимать:
  - Сколько он тратит сейчас.
  - Куда именно уходят деньги (по сервисам/категориям).
  - Что он может сделать прямо сейчас, чтобы платить меньше (явные CTA).
  - Что сервис безопасно работает с его данными и ничего сам не списывает.
