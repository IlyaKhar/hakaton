# OAuth Flow для подключения почты

## Как это работает

### 1. Пользователь выбирает провайдера на фронте

**Экран:** "Подключите источники"

Пользователь видит карточки:
- Mail.ru
- Яндекс.Почта  
- Gmail

**Действие:** Пользователь нажимает на карточку (например, Mail.ru)

---

### 2. Фронт запрашивает URL авторизации

**API запрос:**
```
GET /api/v1/sources/oauth/mailru/authorize
Authorization: Bearer {jwt_token}
```

**Ответ:**
```json
{
  "authorizationUrl": "https://oauth.mail.ru/login?client_id=...&redirect_uri=...&state=..."
}
```

---

### 3. Фронт редиректит пользователя

**Действие:** Фронт открывает `authorizationUrl` в браузере

Пользователь видит страницу авторизации Mail.ru/Yandex/Gmail:
```
┌─────────────────────────────┐
│  Mail.ru                    │
│                             │
│  Войдите в аккаунт          │
│                             │
│  Email:                      │
│  [user@mail.ru       ]       │
│                             │
│  Пароль:                     │
│  [••••••••••••       ]       │
│                             │
│  [  Войти  ]                 │
└─────────────────────────────┘
```

---

### 4. Пользователь авторизуется

**Действие:** Пользователь вводит логин/пароль и нажимает "Войти"

Провайдер проверяет данные и редиректит обратно на наш callback:
```
https://наш-домен.com/api/v1/sources/oauth/mailru/callback?code=ABC123&state=XYZ789
```

---

### 5. Бэкенд обрабатывает callback

**API запрос (автоматически от провайдера):**
```
GET /api/v1/sources/oauth/mailru/callback?code=ABC123&state=XYZ789
```

**Логика бэкенда:**
1. Извлекает `userID` из `state`
2. Обменивает `code` на `access_token`
3. Получает `email` пользователя через API провайдера
4. Создаёт источник со статусом `"connected"`
5. Сохраняет `access_token` в `meta`

**Ответ:**
```json
{
  "message": "Почта успешно подключена",
  "source": {
    "id": "...",
    "type": "email",
    "provider": "mailru",
    "status": "connected",
    "meta": {
      "email": "user@mail.ru",
      "accessToken": "...",
      "refreshToken": "..."
    }
  },
  "redirect": "https://наш-домен.com/sources?connected=..."
}
```

---

### 6. Фронт обрабатывает успешное подключение

**Действие:** Фронт получает ответ и:
- Показывает уведомление: "Почта успешно подключена"
- Редиректит на главную страницу
- Обновляет список источников
- Автоматически запускает первую синхронизацию

## Настройка OAuth приложений

### Mail.ru

1. Зайди в https://oauth.mail.ru/app
2. Создай новое приложение
3. Укажи:
   - **Redirect URI**: `https://твой-домен.com/api/v1/sources/oauth/mailru/callback`
   - **Права**: `userinfo`, `mail.imap`
4. Получи `Client ID` и `Client Secret`

### Yandex

1. Зайди в https://oauth.yandex.ru/
2. Создай новое приложение
3. Укажи:
   - **Redirect URI**: `https://твой-домен.com/api/v1/sources/oauth/yandex/callback`
   - **Права**: `mail.imap`
4. Получи `Client ID` и `Client Secret`

### Gmail (Google)

1. Зайди в https://console.cloud.google.com/
2. Создай новый проект
3. Включи Gmail API
4. Создай OAuth 2.0 credentials
5. Укажи:
   - **Redirect URI**: `https://твой-домен.com/api/v1/sources/oauth/gmail/callback`
   - **Scopes**: `https://www.googleapis.com/auth/gmail.readonly`
6. Получи `Client ID` и `Client Secret`

## Переменные окружения

Добавь в `.env`:

```env
# OAuth настройки для почтовых провайдеров
MAILRU_CLIENT_ID=твой_client_id
MAILRU_CLIENT_SECRET=твой_client_secret

YANDEX_CLIENT_ID=твой_client_id
YANDEX_CLIENT_SECRET=твой_client_secret

GMAIL_CLIENT_ID=твой_client_id
GMAIL_CLIENT_SECRET=твой_client_secret

# Base URL для callback (должен совпадать с Redirect URI в настройках OAuth)
BASE_URL=https://твой-домен.com
```

---

## Безопасность

1. **State параметр**: Содержит закодированный `userID` для проверки
2. **HTTPS обязателен**: OAuth требует HTTPS для callback URL
3. **Токены хранятся зашифрованными**: `accessToken` в `meta` (можно добавить шифрование)
4. **Refresh tokens**: Сохраняются для обновления токенов

---

## Следующие шаги

После успешного подключения:
1. Автоматически запускается первая синхронизация
2. Парсятся письма с чеками/уведомлениями
3. Создаются подписки и транзакции
4. Данные отображаются на главной странице
