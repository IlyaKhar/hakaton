# Автоматическая синхронизация почты через IMAP

## Техническая реализация

### 1. Данные, которые собираем от пользователя:

При подключении почты пользователь указывает:
- **Email** (уже есть)
- **Пароль приложения** (для IMAP доступа)
  - Mail.ru: пароль приложения из настроек безопасности
  - Yandex: пароль приложения из id.yandex.ru/security/app-passwords
  - Gmail: пароль приложения из myaccount.google.com/apppasswords

### 2. Структура данных в `meta`:

```json
{
  "email": "user@mail.ru",
  "imapPassword": "пароль_приложения",
  "lastSyncAt": "2024-03-01T12:00:00Z",  // последняя синхронизация
  "lastMessageId": "12345"  // ID последнего обработанного письма
}
```

### 3. Эндпоинты:

- `POST /sources` - создание источника (уже есть)
  - Теперь в `meta` нужно передавать `email` + `imapPassword`
  
- `POST /sources/:id/sync` - **НОВЫЙ** эндпоинт для ручной синхронизации
  - Подключается к IMAP
  - Ищет новые письма с чеками/уведомлениями
  - Парсит и создаёт транзакции
  - Возвращает количество найденных транзакций

- `GET /sources/:id/status` - **НОВЫЙ** эндпоинт для проверки статуса синхронизации
  - Последняя синхронизация
  - Количество найденных транзакций
  - Ошибки (если есть)

### 4. IMAP настройки для провайдеров:

**Mail.ru:**
- Host: `imap.mail.ru`
- Port: `993` (SSL)
- Username: полный email
- Password: пароль приложения

**Yandex:**
- Host: `imap.yandex.ru`
- Port: `993` (SSL)
- Username: полный email
- Password: пароль приложения

**Gmail:**
- Host: `imap.gmail.com`
- Port: `993` (SSL)
- Username: полный email
- Password: пароль приложения

### 5. Логика парсинга писем:

1. Подключаемся к IMAP
2. Ищем письма за последние 30 дней (или с `lastSyncAt`)
3. Фильтруем письма по:
   - Отправителю (банки, сервисы подписок)
   - Темам ("списание", "оплата", "чек", "receipt", "payment")
   - Вложениям (PDF чеки)
4. Парсим текст письма и вложения
5. Извлекаем транзакции (дата, сумма, сервис)
6. Создаём подписки и транзакции

## UI/UX Flow на фронте

### Экраны и шаги:

#### 1. Экран выбора источника (уже есть в дизайне)

**Экран:** "Подключите источники"

Пользователь видит:
- Секция "Банки" (заглушка, неактивна)
- Секция "Почта" с тремя карточками:
  - Mail.ru (можно выбрать)
  - Яндекс.Почта (можно выбрать)
  - Gmail (можно выбрать)

**Действие:** Пользователь нажимает на карточку почты (например, Mail.ru)

```

│  2. Безопасность → Пароли       │

```

**Действие:** Пользователь нажимает "Подключить"

**API запрос:**
```json
POST /api/v1/sources

  "provider": "mailru",
  "meta": {

    "imapPassword": "пароль_приложения"
  }

```

**Ответ:** Источник создан со статусом `"pending"`

## Автоматическая синхронизация (опционально, для будущего)

### Фоновый процесс:

1. **Cron job** (каждые 6 часов):
   - Находит все источники со статусом `"connected"`
   - Вызывает синхронизацию для каждого
   - Обновляет `lastSyncAt` в `meta`

2. **Webhook/уведомления** (если провайдер поддерживает):
   - Gmail: Google Pub/Sub
   - Yandex: Webhook API
   - Mail.ru: (не поддерживает, только IMAP)

### Уведомления пользователю:

После автоматической синхронизации:
- Push-уведомление: "Найдено 3 новые транзакции"
- Или просто обновляем данные на главной при открытии

## Безопасность

### Хранение паролей:

- **НЕ храним** пароли в открытом виде
- Шифруем `imapPassword` перед сохранением в БД
- Используем AES-256 шифрование
- Ключ шифрования в переменных окружения

### Альтернатива (более безопасно):

- Использовать OAuth вместо паролей
- Но для MVP/хакатона IMAP проще

## Итого:

1. **Пользователь выбирает почту** → экран выбора источника
2. **Вводит email + пароль** → экран подключения
3. **Проверка подключения** → автоматическая синхронизация
4. **Главная страница** → список подписок + кнопка синхронизации
5. **Ручная синхронизация** → по требованию пользователя
