## 1. Персоны (для контекста дизайна и жюри)

- **P1 — Обычный пользователь подписок**
  - 22–40 лет, активно пользуется стримингами, музыкой, софтом по подписке.
  - Боль: не понимает, сколько в сумме улетает денег и за что именно.
  - Цель: увидеть все подписки в одном месте, сократить лишние траты.

- **P2 — Экономный пользователь**
  - 25–45 лет, заморачивается на бюджете.
  - Боль: боится «скрытых» списаний, забытых подписок и повышения цен.
  - Цель: получать умные уведомления заранее и быстро отключать лишнее.

- **P3 — Продвинутый технарь**
  - 20–35 лет, много сервисов, софта, платных тулзов.
  - Боль: сложно всё контролировать руками, хочется автоматизации.
  - Цель: максимум автоматического сбора данных, минимум ручного ввода.

## 2. Юзер-флоу (в формате для диаграмм)

Ниже — описание основных потоков. Это можно переложить в BPMN/flowchart (Figma, Miro) 1-в-1.

### UF-01. Первый запуск и онбординг

1. **Старт** → пользователь открывает приложение первый раз.
2. **Экран онбординга 1** → коротко: «Соберём все подписки в одном месте».
3. **Экран онбординга 2** → «Покажем, сколько ты реально тратишь и где можно сэкономить».
4. **Экран онбординга 3** → «Автоматический анализ платежей и чеков (по согласию)».
5. **Кнопка `Продолжить`** → переход к регистрации/логину.
6. **Выбор способа входа**: телефон / почта / Google / Apple.
7. **Подтверждение** (код, OAuth и т.п.).
8. **Экран согласий**: политика, доступ к данным, пояснение что сервис не списывает деньги сам.
9. **Кнопка `Согласен`** → переход к UF-02 (подключение источников).

### UF-02. Подключение источников (банки и почта)

1. **Экран «Подключите источники»**:
   - Блок `Банки`.
   - Блок `Почта`.
   - Кнопка `Пропустить` (можно перейти к ручному вводу).
2. Пользователь выбирает банк (например, Сбербанк) → тап по карточке.
3. **Экран описания интеграции с банком**:
   - Что читаем: история операций, регулярные платежи.
   - Что не делаем: не списываем деньги, не меняем настройки.
4. **Кнопка `Подключить банк`** → редирект в OAuth/веб.
5. **Успешный возврат** → показываем `Статус: Подключено`.
6. Фон: наличие процесса `Идёт первичный анализ транзакций` (можно индикатор).
7. Аналогично для почты: Mail.ru / Yandex / Gmail → OAuth → статус.
8. После подключения хотя бы одного источника → кнопка `Перейти к моим подпискам` → UF-03.

### UF-03. Просмотр дашборда подписок

1. Пользователь попадает на **главный экран `Мои подписки`**.
2. Если данные ещё грузятся → skeleton/loader + текст `Ищем ваши подписки по операциям и письмам`.
3. После загрузки:
   - Вверху summary: количество подписок, траты в месяц/год.
   - Диаграмма по категориям.
   - Список подписок (карточки).
4. Пользователь:
   - Скроллит список.
   - Может отфильтровать по категории.
   - Может воспользоваться поиском.
5. Тап по карточке конкретной подписки → переход к UF-04 (деталь).

### UF-04. Детали подписки и действия с ней

1. Пользователь на экране конкретной подписки.
2. Видит:
   - Название, логотип, категорию.
   - Стоимость и период.
   - Дату следующего списания.
   - Историю списаний.
   - Блок использования (данные, которые он сам отмечал).
3. Ниже — `Рекомендации`:
   - Альтернативы и возможная экономия.
   - Кнопка `Сравнить тарифы`.
4. Внизу — CTA-кнопки:
   - `Отменить подписку`.
   - `Приостановить`.
   - Дополнительно: `Изменить тариф`.
5. Варианты действий:
   - Нажимает `Отменить подписку` → UF-08.
   - Нажимает `Приостановить` → UF-09.
   - Нажимает `Сравнить тарифы` → UF-07.

### UF-05. Аналитика и прогноз трат

1. Пользователь переключается на вкладку `Аналитика`.
2. Видит:
   - Карточку `Траты за месяц`.
   - Карточку `Траты за год`.
   - График трат по месяцам.
   - Разбивку по категориям и сервисам.
3. Сверху или снизу — блок `Прогноз на год`:
   - Общая сумма прогнозируемых трат.
   - Блок «потенциальная экономия», если отключить/приостановить малополезные подписки.
4. Пользователь может:
   - Менять период.
   - Тапнуть по категории, чтобы провалиться в список подписок этой категории.
   - Тапнуть по подсказке «Где можно сэкономить» → фильтр на подписки с низкой активностью.

### UF-06. Настройка и работа с уведомлениями

1. Пользователь открывает вкладку/экран `Уведомления`.
2. Видит список типов уведомлений:
   - Предстоящее списание.
   - Повышение цены.
   - Низкая активность.
   - Выгодные альтернативы.
3. Для каждого типа:
   - Тумблер вкл/выкл.
   - Настройка каналов (Push, Email, Telegram).
4. При первом включении типа показываем поясняющую подсказку (что именно будет приходить и как часто).
5. Пользователь сохраняет настройки → на бэке обновляются предпочтения.
6. Отдельный экран `История уведомлений`:
   - Список с возможностью пометить как прочитанное / удалить.

### UF-07. Поиск альтернатив и сравнение

1. Пользователь:
   - Либо заходит во вкладку `Рекомендации`.
   - Либо из детали подписки жмёт `Сравнить тарифы`.
2. Видит:
   - Список альтернативных сервисов в той же категории.
   - Фильтры по цене, региону, типу тарифа.
3. При выборе сервиса:
   - Открывается экран сравнения: текущий сервис vs альтернатива.
   - Табличное сравнение цен, ограничений и плюсов/минусов.
4. Кнопка `Перейти на сайт сервиса` → внешний браузер.

### UF-08. Быстрая отмена подписки

1. Пользователь в деталях подписки нажимает `Отменить подписку`.
2. Открывается модалка:
   - Краткое пояснение, что сервис не может гарантировать отмену у провайдера.
   - Варианты:
     - `Открыть страницу управления подпиской` (если известен URL).
     - `Создать письмо в поддержку`.
3. При выборе `Страница управления`:
   - Открываем ссылку.
   - В приложении подписка помечается как `В процессе отмены`.
4. При выборе `Письмо`:
   - Генерируем шаблон письма (тема + текст).
   - Открываем почтовый клиент.
   - В приложении — также статус `В процессе отмены`.
5. Пользователь может вручную затем изменить статус на `Отменена` (например, чекбокс «Я подтвердил отмену»).

### UF-09. Приостановка подписки (пауза)

1. Пользователь жмёт `Приостановить`.
2. Модалка:
   - Выбор периода паузы (1/3 месяца, до даты).
   - Выбор типа:
     - `Только в аналитике` (виртуальная пауза).
     - `Помочь приостановить у сервиса` (аналогично UF-08: ссылка/письмо).
3. Если пользователь выбирает `Только в аналитике`:
   - Статус подписки в приложении → `Приостановлена`.
   - В прогнозе трат на период паузы эта подписка не учитывается.
4. Если `Помочь приостановить`:
   - Открываем ссылку или формируем письмо по шаблону.
   - Статус → `В процессе приостановки`.

### UF-10. Ручное добавление подписки

1. Пользователь жмёт `Добавить подписку` (FAB / кнопка).
2. Открывается форма:
   - Название (автодополнение).
   - Категория.
   - Периодичность.
   - Цена.
   - Дата следующего списания.
   - Источник оплаты.
3. Кнопка `Сохранить`.
4. Подписка появляется в списке и учитывается в аналитике/прогнозе.

## 3. Юзерстори (для ТЗ / презентации)

Формат: «Как [кто], я хочу [что], чтобы [зачем]».

### Базовые

- **US-01. Как пользователь, я хочу увидеть все свои подписки в одном месте, чтобы понимать общую сумму трат.**
- **US-02. Как пользователь, я хочу подключить банк и почту, чтобы сервис сам находил мои подписки по операциям и чекам.**
- **US-03. Как пользователь, я хочу получать уведомления о предстоящих списаниях, чтобы успевать отменить ненужные подписки.**
- **US-04. Как пользователь, я хочу видеть аналитику по категориям и сервисам, чтобы понимать, на что уходит больше всего денег.**
- **US-05. Как пользователь, я хочу прогноз трат на год, чтобы оценить, сколько мне обойдутся все подписки в долгосроке.**

### Управление подписками

- **US-06. Как пользователь, я хочу быстро отменить подписку через переход в личный кабинет или письмо в поддержку, чтобы не тратить время на поиск инструкций.**
- **US-07. Как пользователь, я хочу приостанавливать подписку на время, чтобы временно снизить расходы без полного отказа.**
- **US-08. Как пользователь, я хочу отмечать, пользуюсь ли я подпиской, чтобы сервис мог подсказать, какие из них лишние.**

### Рекомендации и экономия

- **US-09. Как пользователь, я хочу видеть альтернативные, более выгодные сервисы, чтобы экономить при сохранении нужного функционала.**
- **US-10. Как пользователь, я хочу сравнивать текущую подписку с альтернативами по цене и условиям, чтобы осознанно менять сервис.**

### Уведомления и контроль

- **US-11. Как пользователь, я хочу настраивать типы уведомлений и каналы (Push/Email/Telegram), чтобы не получать лишнего спама и не пропускать важное.**
- **US-12. Как пользователь, я хочу получать уведомления о повышении цены, чтобы вовремя решить, оставаться ли на сервисе.**
- **US-13. Как пользователь, я хочу получать уведомления о редком использовании сервиса, чтобы отключать неиспользуемые подписки.**

### Прозрачность и доверие

- **US-14. Как пользователь, я хочу чётко понимать, какие данные читает приложение (банки, почта), чтобы не переживать за безопасность.**
- **US-15. Как пользователь, я хочу быть уверен, что приложение само ничего не списывает, чтобы спокойно подключать свои карты и аккаунты.**

## 4. Как это можно презентовать жюри

- **Слайд 1 — Проблема**: люди теряют деньги на незаметных и забытых подписках.
- **Слайд 2 — Решение**: единая платформа (мобилка + веб), которая собирает все подписки, считает, анализирует и подсказывает, где сэкономить.
- **Слайд 3 — Ключевые фичи**: автоматический сбор, аналитика, прогноз, умные уведомления, быстрые отмены/паузa, рекомендации альтернатив.
- **Слайд 4 — Юзер-флоу**: 1) онбординг → 2) подключение источников → 3) дашборд → 4) аналитика и прогноз → 5) действия по экономии.
- **Слайд 5 — Ценность**: «сколько денег пользователь может сэкономить в год» + примеры сценариев.
